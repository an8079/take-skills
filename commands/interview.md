# /interview - 需求访谈

启动需求访谈流程，深入理解项目需求和用户期望。

## 使用方式

```
/interview
```

或

```
开始访谈
```

## 工作流程

1. 调用 **interviewer agent**
2. 按照 8 维度进行访谈
3. 识别是否需要创建新技能
4. 完成后输出访谈摘要

## 访谈维度

| 维度 | 说明 |
|------|------|
| 业务理解 | 要解决什么问题、为谁解决、为什么现有方案不行 |
| 功能边界 | MVP 范围、哪些不做、边界情况 |
| 技术可行性 | 技术栈、架构方向、技术难点 |
| 数据流 | 数据从哪来、存到哪、怎么流转 |
| 接口契约 | 对外接口、输入输出 |
| 异常处理 | 错误场景、处理策略 |
| 非功能需求 | 性能、安全、可维护性 |
| 部署交付 | 部署方式、环境要求、交付物 |

## 快速模式

如果需求已经明确，可以使用快速模式：

```
快速开始
跳过访谈
```

## 访谈完成后

确认访谈内容后，使用以下命令进入下一阶段：

```
/spec      # 生成规格文档
/plan      # 直接进入计划阶段（跳过详细规格）
```

---

**提示：** 访谈不按固定轮次结束，而是根据完整度判断。所有维度满足后才会结束。
