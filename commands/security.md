# /security - 安全审查

对代码进行全面的安全审计。

## 使用方式

```
/security
```

或

```
安全审查
安全审计
```

## 前置条件

- 代码审查已完成
- 所有 Critical 和 Important 问题已处理

## 审查范围（OWASP Top 10）

| 类别 | 检查项 |
|------|--------|
| A01 访问控制失效 | 水平越权、垂直越权、IDOR |
| A02 加密失效 | 硬编码密钥、弱加密、明文密码 |
| A03 注入 | SQL 注入、命令注入、NoSQL 注入 |
| A04 不安全设计 | 缺少安全设计、默认配置 |
| A05 安全配置错误 | 暴露调试信息、默认账户 |
| A06 易受攻击组件 | 过时的依赖、不再维护的包 |
| A07 身份验证失败 | 弱密码策略、密码未哈希 |
| A08 数据完整性失效 | 未验证依赖、未验证数据 |
| A09 日志监控失败 | 未记录安全事件、敏感信息日志 |
| A10 SSRF | 可控制的 URL 请求 |

## 审查流程

1. 调用 **security-reviewer agent** 进行审计
2. 扫描常见安全漏洞模式
3. 检查依赖漏洞
4. 分析敏感信息泄露风险
5. 生成安全报告

## 问题严重级别

| 级别 | 含义 | 处理方式 |
|------|------|----------|
| **Critical** | 严重安全漏洞 | 必须立即修复 |
| **High** | 高风险安全问题 | 必须尽快修复 |
| **Medium** | 中等风险 | 建议修复 |
| **Low** | 低风险 | 可选修复 |

## 依赖安全检查

```bash
npm audit
npm audit fix
yarn audit
pip-audit
```

## 安全审查通过

- 无 Critical 级别问题
- High 级别问题已修复或计划修复
- Medium/Low 问题有处理方案

## 安全审查通过后的下一步

```
/build         # 构建项目
/deploy        # 部署项目
```

---

**提示：** 安全是不可妥协的要求。一个漏洞可能导致严重的数据泄露和经济损失。
