# /build - 构建项目

构建项目并生成可部署的产物。

## 使用方式

```
/build
```

或

```
构建项目
执行构建
```

## 前置条件

- 所有测试通过
- 代码审查和安全审查通过

## 构建流程

1. 调用 **devops agent** 处理构建
2. 安装依赖
3. 运行类型检查
4. 运行 Lint 检查
5. 执行构建命令
6. 生成构建报告

## 构建步骤

| 步骤 | 命令 | 说明 |
|------|------|------|
| 安装依赖 | `npm ci` | 清理后安装 |
| 类型检查 | `npm run type-check` | TypeScript 检查 |
| Lint 检查 | `npm run lint` | 代码风格检查 |
| 运行测试 | `npm test` | 确保测试通过 |
| 构建产物 | `npm run build` | 生成 dist/ |

## 构建产物

| 产物 | 位置 | 说明 |
|------|------|------|
| 源代码编译 | `dist/` | TypeScript/JSX 编译结果 |
| 静态资源 | `dist/static/` | 图片、字体等 |
| 包文件 | `package.tgz` | 可分发的包 |
| Docker 镜像 | `username/app:tag` | 容器镜像 |

## 构建检查清单

- [ ] 类型检查通过
- [ ] Lint 检查通过
- [ ] 所有测试通过
- [ ] 构建成功
- [ ] 产物大小合理
- [ ] 环境变量正确配置

## 构建成功后的下一步

```
/deploy        # 部署项目
/package       # 打包交付
```

---

**提示：** 构建失败时，首先检查错误信息，确保所有依赖正确安装。
