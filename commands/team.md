# /team - 动态专家组队

根据任务需求动态组建专家团队，调用多个专家 Agent 协同工作。

## 使用方式

```
/team "任务描述"
```

或

```
组队
组建团队
专家团队
```

## 团队角色

### 核心角色

| 角色 | emoji | 专长 |
|------|-------|------|
| **软件架构师** | 🏛️ | 系统设计、DDD、架构模式 |
| **高级开发者** | 💎 | Laravel、React/Vue、Three.js |
| **代码审查员** | 👁️ | 代码审查、质量保证 |
| **安全工程师** | 🔒 | 威胁建模、漏洞评估、安全架构 |

### 扩展角色

| 角色 | emoji | 专长 |
|------|-------|------|
| **后端架构师** | 🏗️ | 数据库、API、云基础设施 |
| **前端开发者** | 🖥️ | React/Vue、响应式设计、性能优化 |
| **DevOps 自动化** | ⚙️ | CI/CD、容器化、基础设施即代码 |
| **数据工程师** | 📊 | ETL、数据管道、数据建模 |
| **技术作家** | 📝 | API 文档、技术规格、用户文档 |
| **SRE** | 📈 | 监控、告警、可靠性、事件响应 |

## 团队配置模板

### 小型团队（2-3人）
适用于简单任务或 MVP。

```
/team "构建用户认证系统"
→ 软件架构师 + 安全工程师
```

### 中型团队（3-4人）
适用于标准项目功能。

```
/team "构建电商后端"
→ 后端架构师 + 安全工程师 + 代码审查员
```

### 大型团队（5+人）
适用于复杂系统或需要多领域专家。

```
/team "构建完整 SaaS 平台"
→ 软件架构师 + 后端架构师 + 前端开发者 + 安全工程师 + DevOps
```

## 工作流程

```
用户输入任务
     ↓
分析任务需求
     ↓
推荐团队配置
     ↓
用户确认/调整
     ↓
激活专家 Agent
     ↓
专家协作执行
     ↓
汇总结果报告
```

## 团队协作模式

### 1. 串行模式
```
架构师 → 开发者 → 审查员
```
适用于：需要严格顺序的任务

### 2. 并行模式
```
架构师 ──→ 开发者 A
    │
    └──→ 开发者 B
           │
           ↓
        审查员
```
适用于：任务可分解、独立执行

### 3. 迭代模式
```
开发者 → 审查员 → 修复 → 审查员 → ...
```
适用于：需要高质量产出的任务

## 用户同意机制

1. **团队配置阶段** - 显示团队配置，用户确认
2. **角色调整阶段** - 用户可增删角色
3. **执行阶段** - 团队执行任务
4. **结果确认** - 用户审核最终结果

## 与 /test-teams 的区别

| 维度 | /team | /test-teams |
|------|-------|-------------|
| 目的 | 完成任务 | 测试验证 |
| 角色 | 动态配置 | 3测试员+1资深 |
| 灵活性 | 完全灵活 | 固定配置 |
| 适用 | 设计、开发、架构 | 测试、QA |
