# /plan - 创建实现计划

将规格文档拆解为可执行的任务清单。

## 使用方式

```
/plan
```

或

```
做计划
创建计划
拆解任务
```

## 前置条件

- 规格文档已完成并确认
- 技术栈已确定

## 工作流程

1. 调用 **planner agent** 拆解任务
2. 识别任务依赖关系
3. 确定实现顺序
4. 识别潜在风险
5. 输出完整任务计划

## 任务计划包含

| 内容 | 说明 |
|------|------|
| 任务列表 | 所有任务的清单，包含 ID、名称、优先级、复杂度 |
| 任务详情 | 每个任务的描述、步骤、产出物、验收标准 |
| 依赖关系图 | 任务之间的依赖关系 |
| 里程碑 | 按阶段组织的任务分组 |
| 风险矩阵 | 潜在风险和应对措施 |

## 任务优先级

| 优先级 | 定义 | 处理策略 |
|--------|------|----------|
| P0 | 核心功能、阻塞其他任务 | 必须优先完成 |
| P1 | 重要功能、影响用户体验 | 尽早完成 |
| P2 | 优化功能、锦上添花 | 有空再做 |

## 确认后的下一步

```
/code      # 开始编码实现
```

---

**提示：** 好的计划是成功的基础。确保任务拆分合理、依赖关系清晰。
