# /debug - 调试模式

启动调试流程，分析错误并提供修复建议。

## 使用方式

```
/debug
```

或

```
开始调试
```

## 调试流程

1. **分析错误**
   - 读取错误日志
   - 分析堆栈跟踪
   - 定位问题代码

2. **根因分析**
   - 识别问题的根本原因
   - 分析相关代码逻辑
   - 检查配置和依赖

3. **提供修复建议**
   - 给出具体的修复方案
   - 提供代码示例
   - 建议验证步骤

## 调试技巧

- 提供完整的错误信息
- 描述你尝试过的解决方案
- 说明当前环境（操作系统、版本等）

## 调试后

```
/code      # 修复后继续编码
/test      # 运行测试验证
/review    # 代码审查
```

## 示例

```
/debug
错误信息: TypeError: Cannot read property 'map' of undefined
堆栈: at UserList.render (UserList.js:45)
```
