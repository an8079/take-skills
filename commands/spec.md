# /spec - 生成规格文档

根据访谈内容，生成完整的项目规格文档。

## 使用方式

```
/spec
```

或

```
写规格
生成规格文档
```

## 前置条件

- 需求访谈已完成（所有 8 个维度）
- 访谈摘要已确认

## 工作流程

1. 调用 **architect agent** 进行架构设计
2. 调用 **spec-writing skill** 生成规格文档
3. 规格文档输出到 `docs/spec.md`
4. 等待用户确认

## 规格文档包含

| 章节 | 内容 |
|------|------|
| 项目概述 | 背景、目标用户、成功标准、项目边界 |
| 功能规格 | 功能清单、用户故事、验收标准、用户流程 |
| 技术规格 | 技术选型、系统架构、数据模型、接口定义 |
| 非功能需求 | 性能要求、安全要求、可靠性、可维护性 |
| 实现计划 | 任务拆解、任务详情、里程碑 |
| 风险与应对 | 技术风险、待确认事项、技术债务 |
| 部署与交付 | 部署架构、环境要求、交付物清单 |

## 确认后的下一步

```
/plan      # 创建实现计划
/code      # 直接开始编码（跳过详细计划）
```

---

**提示：** 规格文档是开发的基础，确保准确和完整后再进入下一阶段。
