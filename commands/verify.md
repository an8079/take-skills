# /verify - 验证循环

运行完整的验证流程，确保项目质量。

## 使用方式

```
/verify
```

或

```
验证
验证循环
```

## 验证流程

```
1. 运行单元测试
2. 运行集成测试
3. 运行 E2E 测试
4. 生成覆盖率报告
5. 分析测试结果
6. 生成修复建议
7. 重新验证（如需要）
```

## 验证步骤

### 步骤 1：单元测试

```bash
npm test
```

检查点：
- 所有单元测试通过
- 测试覆盖率 >= 80%

### 步骤 2：集成测试

```bash
npm run test:integration
```

检查点：
- 所有集成测试通过
- API 端点响应正常

### 步骤 3：E2E 测试

```bash
npm run test:e2e
```

检查点：
- 所有 E2E 测试通过
- 核心用户流程正常

### 步骤 4：覆盖率报告

```bash
npm run test:coverage
```

检查点：
- 语句覆盖 >= 85%
- 分支覆盖 >= 80%
- 函数覆盖 >= 90%

### 步骤 5：类型检查

```bash
npm run type-check
```

检查点：
- 无类型错误

### 步骤 6：Lint 检查

```bash
npm run lint
```

检查点：
- 无 Lint 错误

## 自动化验证

可以配置为在每次提交前自动运行：

```json
// package.json
{
  "scripts": {
    "precommit": "npm run verify"
  }
}
```

或使用 Husky：

```bash
npm install husky --save-dev
npx husky add .husky/pre-commit "npm run verify"
```

---

**提示：** 持续的验证可以及早发现问题，减少后期的修复成本。
