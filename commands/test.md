# /test - 运行测试

运行项目的测试套件并生成报告。

## 使用方式

```
/test
```

或

```
运行测试
执行测试
```

## 测试类型

| 类型 | 命令 | 说明 |
|------|------|------|
| 单元测试 | `npm test` | 测试函数/类 |
| 集成测试 | `npm run test:integration` | 测试模块交互 |
| E2E 测试 | `npm run test:e2e` | 测试用户流程 |
| 覆盖率 | `npm run test:coverage` | 生成覆盖率报告 |

## 测试执行流程

1. 调用 **tester agent** 生成测试（如果需要）
2. 运行所有测试
3. 分析测试结果
4. 生成测试报告
5. 识别失败的测试

## 测试报告包含

| 内容 | 说明 |
|------|------|
| 测试统计 | 总数、通过、失败、跳过 |
| 覆盖率分析 | 语句、分支、函数覆盖率 |
| 失败测试详情 | 失败原因、修复建议 |
| 覆盖率不足的文件 | 需要补充测试的文件 |

## 测试检查清单

- [ ] 单元测试覆盖率 >= 80%
- [ ] 核心功能有集成测试
- [ ] 关键用户流程有 E2E 测试
- [ ] 边界情况有测试
- [ ] 错误处理有测试

## 测试通过后的下一步

```
/review        # 代码审查
/verify        # 验证循环
```

## 测试失败时的处理

```
/debug         # 调试失败原因
```

---

**提示：** 充分的测试是代码质量的保障。目标是高覆盖率，但更重要的是测试的有效性。
