# /code - 编码模式

进入编码模式，按计划逐步实现功能。

## 使用方式

```
/code
```

或

```
开始编码
编码模式
```

## 前置条件

- 实现计划已确认
- 当前任务已明确

## 工作流程

1. 调用 **coder agent** 实现任务
2. 每次只做一个任务
3. 完成后展示结果
4. 等待用户确认
5. 确认后继续下一个任务

## 编码原则

| 原则 | 说明 |
|------|------|
| 简洁性 | 代码简洁明了，易于理解 |
| 可读性 | 命名有意义，逻辑清晰 |
| 可维护性 | 模块化设计，易于修改 |
| 可测试性 | 代码结构便于测试 |
| 性能考虑 | 在复杂度和性能之间找到平衡 |

## 代码风格

- 遵循项目代码规范
- 函数长度不超过 20-30 行
- 文件大小不超过 800 行
- 深层嵌套不超过 4 层
- 不使用硬编码值

## 每个任务完成后

```
下一个任务      # 继续下一个任务
/test          # 运行测试
/review        # 代码审查
```

---

**提示：** 编码过程中遇到问题，可以调用 debug-helper agent 进行诊断。
