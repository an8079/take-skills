# /reflect - 反思学习

处理学习记录，将用户纠正和经验固化到知识库。

## 使用方式

```
/reflect
```

或

```
反思
反思学习
处理学习记录
```

## 何时使用

| 时机 | 说明 |
|------|------|
| 完成一个功能/任务后 | 如果刚才有纠正我的地方 |
| 用户多次纠正 AI 后 | 建议运行以保存学习 |
| 准备结束会话时 | 处理待学习的内容 |
| 里程碑完成时 | 保存本阶段的学习 |

## 反思流程

1. 读取 `memory-bank/学习记录.md`
2. 展示待处理的纠正
3. 用户确认后应用学习
4. 更新规则文件
5. 标记为已应用

## 纠正检测模式

当用户说以下内容时，自动识别为纠正：

| 模式 | 示例 |
|------|------|
| 否定 + 正确做法 | 「不对，用 X 不是 Y」「别用 A， 用 B」 |
| 实际上... | 「实际上应该...」「其实是...」 |
| 记住... | 「记住：...」「以后要...」 |
| 这样不行 | 「这样不行，应该...」「错了，要...」 |

## 学习应用范围

| 范围 | 应用位置 |
|------|----------|
| 全局 | CLAUDE.md |
| 项目 | memory-bank/学习记录.md 的项目规则 |
| 领域 | 对应的 SKILL.md |

## 反思输出示例

```
🔍 **反思：发现 3 条待处理的纠正**

| # | 用户说的 | AI 学到的 | 建议范围 |
|---|----------|----------|----------|
| 1 | 不对，用 gpt-5.1 不是 gpt-5 | 推理任务用 gpt-5.1 | 全局 |
| 2 | 这个项目用 PostgreSQL | 本项目用 PostgreSQL | 项目 |
| 3 | 别用 var，用 const | JS 中优先用 const | 全局 |

请选择：
- 「全部应用」— 接受所有学习
- 「选择应用」— 逐条确认
- 「跳过」— 暂不处理
```
