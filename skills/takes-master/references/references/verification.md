# 四层验证标准详解

## 概览

GSD 的 `gsd-verifier` 使用四层验证，每层递进，确保交付物真正达标：

```
L1: 存在性 → "东西在不在？"
L2: 实质性 → "东西是不是真的能用？"
L3: 连线   → "东西之间连上了吗？"
L4: 数据流 → "数据跑通了吗？"
```

---

## L1: 存在性验证

**问题**：计划要求的东西都创建了吗？

### 检查清单

```markdown
## 文件结构
- [ ] 所有计划的文件已创建
- [ ] 目录结构符合设计
- [ ] 配置文件就位

## API端点
- [ ] PLAN中列出的每个端点已实现
- [ ] HTTP方法正确（GET/POST/PUT/DELETE）
- [ ] 路径命名符合REST规范

## 功能模块
- [ ] 每个计划功能模块已实现
- [ ] 核心逻辑已编写（非空壳）

## 测试
- [ ] 单元测试文件已创建
- [ ] 集成测试文件已创建（如需要）
```

### 通过标准
- 100%文件存在
- 100%端点实现
- 如有任何遗漏 → 验证失败

---

## L2: 实质性验证

**问题**：每个东西是不是真的能用，不是空壳？

### 检查清单

```markdown
## API实质性
- [ ] 每个端点有实际处理逻辑（不是空return）
- [ ] 输入参数有校验
- [ ] 错误有明确响应（不是静默失败）
- [ ] 数据库操作有实现（不是空函数）

## 业务逻辑实质性
- [ ] 核心算法已实现
- [ ] 边界条件有处理
- [ ] 状态机/流程控制正确

## 前端实质性
- [ ] 组件有实际渲染逻辑
- [ ] 表单有验证
- [ ] 加载/错误状态有处理
- [ ] 不是纯静态页面

## 测试实质性
- [ ] 测试用例有实际断言
- [ ] 不是空测试（只有import和describe）
- [ ] 覆盖核心路径
```

### 通过标准
- 每个端点/函数有实质逻辑
- 核心路径100%覆盖
- 如有空壳 → 验证失败

---

## L3: 连线验证

**问题**：各模块之间的接口调用正确连接了吗？

### 检查清单

```markdown
## API集成
- [ ] 前端调用的URL与后端实现的URL匹配
- [ ] 请求/响应格式一致
- [ ] 认证/授权正确传递

## 模块依赖
- [ ] 计划的模块依赖关系已建立
- [ ] 没有循环依赖（除非计划中明确）
- [ ] 接口契约正确实现

## 数据库
- [ ] 模型与表结构匹配
- [ ] 迁移脚本正确
- [ ] 外键关系正确

## 第三方服务
- [ ] API Key/凭证配置正确
- [ ] 调用端点正确
- [ ] 错误处理完备
```

### 通过标准
- 所有计划中的连接都已建立
- 无意外断裂
- 如有断开 → 验证失败

---

## L4: 数据流验证

**问题**：数据是否按预期正确流转？

### 检查清单

```markdown
## 输入→处理→输出
- [ ] 用户输入能正确接收
- [ ] 输入被正确存储/处理
- [ ] 输出被正确返回/渲染

## 数据完整性
- [ ] 写入的数据能正确读出
- [ ] 更新操作正确生效
- [ ] 删除操作正确移除数据

## 端到端场景
- [ ] 关键用户路径全链路测试通过
- [ ] 登录→操作→登出 流程正确
- [ ] 创建→读取→更新→删除 流程正确

## 性能
- [ ] 响应时间在可接受范围
- [ ] 无内存泄漏
- [ ] 无无限循环
```

### 通过标准
- 所有关键数据流路径验证通过
- E2E场景100%通过
- 如有路径不通 → 验证失败

## 验证报告模板

```markdown
# 验证报告：阶段[N]

## 验证时间：[时间]
## 验证人：gsd-verifier

## L1: 存在性 ✅/❌
- 通过项：[N项]
- 失败项：[如有]

## L2: 实质性 ✅/❌
- 通过项：[N项]
- 失败项：[如有]

## L3: 连线 ✅/❌
- 通过项：[N项]
- 失败项：[如有]

## L4: 数据流 ✅/❌
- 通过项：[N项]
- 失败项：[如有]

## 最终结论
✅ **通过** — 可进入下一阶段
❌ **失败** — 需要修复后重验
⚠️ **有警告** — 通过但需关注警告项

## 失败项详情
| 层级 | 问题 | 修复建议 | 优先级 |
|------|------|---------|--------|
| L2 | API `/auth/login` 无错误处理 | 添加try-catch和错误响应 | P1 |
```
