# self-iterating

**自主迭代循环 Skill** — 通过 Stop Hook 实现持续迭代直到完成任务。

## 触发条件

当用户说：
- 「开始迭代循环」
- 「自主迭代执行 [任务]」
- 「迭代直到 [完成条件]」
- `/iterate [任务] --until [条件] --max [次数]`

## 输入格式

```
迭代任务：[任务描述]
完成条件：[成功标志，如 "测试通过"、"输出包含 XXX"、"无错误"等]
最大迭代次数：[数字，默认 10]
```

## 工作流

```
┌─────────────────────────────────────────────────────────────┐
│                     迭代循环工作流                            │
├─────────────────────────────────────────────────────────────┤
│  1. 解析任务                                                 │
│     ├── 提取任务描述                                         │
│     ├── 提取完成条件                                         │
│     └── 提取最大迭代次数                                     │
│                                                             │
│  2. 执行迭代循环 (最多 N 次)                                 │
│     │                                                       │
│     ├─→ [执行任务] ─→ [检查完成条件] ─→ [是] → [结束]       │
│     │        ↑                      ↓                        │
│     │       [否] ──────────→ [注入状态 → 重新执行]           │
│     │                                                       │
│     └── [达到最大迭代] → [报告失败/需要人工介入]             │
│                                                             │
│  3. 输出结果                                                 │
│     ├── 成功：最终状态 + 迭代次数                             │
│     ├── 失败：失败原因 + 尝试次数                             │
│     └── 终止：达到最大迭代次数                               │
└─────────────────────────────────────────────────────────────┘
```

## 迭代状态

每次迭代后会收集以下状态：

| 状态项 | 来源 | 说明 |
|--------|------|------|
| `git_diff` | `git diff` | 本次迭代的代码变更 |
| `git_log` | `git log -3` | 最近 3 次提交 |
| `test_output` | 测试命令输出 | 测试结果 |
| `last_error` | stderr/日志 | 最后一次错误 |
| `iteration_count` | 计数器 | 当前迭代次数 |
| `files_changed` | `git status` | 已修改文件列表 |

## 完成条件类型

| 类型 | 示例 | 检测方式 |
|------|------|----------|
| 关键字 | "输出包含 SUCCESS" | 字符串匹配 |
| 测试通过 | "测试通过" | 退出码 = 0 |
| 文件存在 | "文件存在 /path/to/file" | 文件检查 |
| 组合 | "测试通过 且 输出包含 OK" | 逻辑组合 |

## 工具链

### 核心脚本

- `scripts/self-iterate/iterate.sh`: 主迭代循环脚本
- `scripts/self-iterate/get-state.sh`: 状态收集脚本
- `scripts/self-iterate/stop-hook.sh`: Stop Hook 拦截脚本

### 使用方式

```bash
# 基本用法
./scripts/self-iterate/iterate.sh "修复 bug" "测试通过" 5

# 复杂条件
./scripts/self-iterate/iterate.sh "实现功能" "输出包含 OK 且 测试通过" 10
```

## 输出格式

### 成功
```
✅ 任务完成
迭代次数: 3/10
完成条件: 测试通过
最终状态: [摘要]
```

### 失败
```
❌ 任务失败
迭代次数: 10/10
失败原因: [原因]
需要人工介入: true
```

### 达到最大迭代
```
⚠️ 达到最大迭代次数
迭代次数: 10/10
当前进度: [进度描述]
建议: [下一步建议]
```

## 质量门禁

每次迭代必须：
1. **记录状态** — 每次迭代后保存状态快照
2. **检查终止** — 满足条件立即终止，不多余执行
3. **日志追踪** — 记录每次迭代的输入输出
4. **安全上限** — 达到最大迭代必须停止

## 注意事项

- Stop Hook 需要 Claude Code 支持 hooks 配置
- 迭代过程中模型可以读取任何文件
- 建议设置合理的最大迭代次数（默认 10）
- 复杂任务建议拆分为多个小任务迭代
