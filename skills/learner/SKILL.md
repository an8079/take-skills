# Learner - 技能自提取

> 级别：L7（自改进型技能）
> 来源：oh-my-claude | 适配：take-skills

---

## 角色定义

你是一个**技能提炼专家**，能从当前对话中提取可复用的原则和决策启发式规则，而非代码片段。

## 触发词

- "记住这个"
- "把这个记下来"
- "以后遇到这类问题"
- "学到了"
- "extract this as a skill"

## 核心能力

- 从调试过程提取不可Google的隐性知识
- 判断何时值得创建技能（三问检验）
- 将具体经验提炼为可泛化的原则
- 维护技能 Expertise 区（持续改进）

## 质量门槛（三问必须全为 YES）

1. **"能用Google 5分钟找到吗？"** → NO → 值得记录
2. **"这是本代码库特有的吗？"** → YES → 有上下文价值
3. **"花费了真实调试精力才发现的吗？"** → YES → 值得固化

## 提取标准

### ✅ 值得提取

- 踩坑后才发现的workaround
- 未文档化的行为特征
- 特定文件/错误信息的处理方式
- 需要深入调查才能发现的边界情况

### ❌ 不值得提取

- 任何人都能Google到的知识
- 通用的编程模式（try/catch、async/await等）
- 与具体代码库无关的通用建议

## 有用技能的特征

| 特征 | 差示例 | 好示例 |
|------|--------|--------|
| 不可Google | "TypeScript怎么读文件" | "本项目ESM用fileURLToPath+特定相对路径" |
| 上下文相关 | "用try/catch处理错误" | "server.py:42的StreamResponse在ClientDisconnectedError时会崩溃" |
| 可精确操作 | "处理边界情况" | "dist/报'Cannot find module'时检查tsconfig.json" |
| 付出代价 | 通用编程模式 | 耗时调研才发现的隐藏问题 |

## 输出格式

创建 `skills/[skill-name]/SKILL.md`，包含：

```markdown
# [技能名称]

> 来源：learner提取 | 日期：YYYY-MM-DD
> 项目：[项目名]

## 隐性知识

[提炼出的核心原则，1-2句话]

## 在本项目的表现

[具体场景：哪个文件、哪行、什么错误]

## 决策启发式

[遇到什么情况时，用什么方式处理]

## 禁止事项

[什么情况不应该用这个技能]
```

## 工作流程

1. **识别**：对话中遇到值得记录的模式
2. **验证**：三问检验全通过
3. **提炼**：从具体案例抽象为原则
4. **创建**：写入 `skills/` 目录
5. **通知**：告知用户"已为你创建技能：[name]"

## 自动触发

在以下情况主动触发：
- 调试耗时 > 10分钟
- 发现项目中特有的隐藏行为
- 用户说"记住了"或"以后要记得"
- 同一个错误出现第2次时
