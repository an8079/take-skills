# Prompt Engineering Guide

## 提示词组成要素

### 1. 角色设定 (Role)
- 指定AI扮演的专业身份
- 示例：`你是一个资深后端工程师`，`你是一个产品经理`

### 2. 背景信息 (Context)
- 项目的现有状态
- 技术栈/工具限制
- 业务场景和约束

### 3. 任务目标 (Task)
- 用动词开头，明确最终产出
- 不说"帮我学习X"，说"给我解释X的核心概念"

### 4. 输出要求 (Requirements)
- 格式要求（JSON、Markdown、代码等）
- 长度要求
- 质量标准

### 5. 参考材料 (References)
- 粘贴相关代码、文档、示例
- 帮助AI理解上下文

## 场景化模板

### 代码修改
```
角色：你是一个[语言]资深工程师
背景：我正在开发[项目]，使用[技术栈]
任务：帮我优化以下代码的性能/可读性/安全性
代码：[粘贴代码]
要求：
- 保持原有功能不变
- 解释你做了哪些改动及原因
```

### 项目构建
```
角色：你是一个全栈架构师
背景：我需要搭建一个[项目类型]，目标用户是[用户群体]
约束：技术栈偏好[xxx]，上线时间是[xxx]
任务：
1. 给出技术方案选择建议
2. 给出项目目录结构
3. 给出核心代码实现
要求：代码可运行，遵循最佳实践
```

### 报告撰写
```
角色：你是一个[行业]行业分析师
背景：[公司/产品/市场]的[现状描述]
任务：撰写一份[报告类型]
要求：
- 结构清晰，使用多级标题
- 每个论点有数据支撑
- 包含执行建议
风格：专业、简洁、有洞见
```

## 常见错误

| 错误 | 修正 |
|------|------|
| "帮我看看这个代码" | "这个代码有什么性能问题？如何优化？" |
| "写个项目" | "写一个Todo App，使用React+Node.js，需要用户认证" |
| "帮我写报告" | "帮我写一份Q1季度产品运营报告，突出用户增长和留存数据" |

## 进阶技巧

### Few-shot 示例
在请求中加入1-3个示例，帮助AI理解你期望的输出格式

### 链式思考 (Chain of Thought)
加一句"请分步骤思考"，让AI展示推理过程

### 限制范围
明确说"不要包含X"，减少无关输出

### 指定语气
"用专业但易懂的语言" vs "用口语化的方式解释"
