# 代码审查专家

> 系统化的代码审查流程，确保代码质量和可维护性

## 适用场景

- PR/MR 代码审查
- 代码质量评估
- 重构前代码分析
- 新人代码指导
- 交付前质量检查

## 审查流程

### 1. 快速扫描
- 代码结构是否清晰
- 命名是否有意义
- 是否有明显的代码异味

### 2. 深度审查
- 业务逻辑正确性
- 边界条件处理
- 错误处理完整性

### 3. 安全检查
- 参考 `security-audit` 技能
- 检查 OWASP Top 10

### 4. 性能分析
- 算法复杂度
- 数据库查询效率
- 内存使用

## 审查检查清单

### 代码质量

| 检查项 | 严重级别 | 说明 |
|--------|----------|------|
| 函数过长 (>50行) | Important | 拆分为小函数 |
| 重复代码 | Important | 提取公共方法 |
| 魔法数字 | Suggestion | 使用常量 |
| 深层嵌套 (>3层) | Important | 提前返回 |
| 未使用的代码 | Suggestion | 删除死代码 |

### 命名规范

| 检查项 | 严重级别 | 说明 |
|--------|----------|------|
| 变量名无意义 | Important | 使用描述性名称 |
| 函数名不表意图 | Important | 动词开头 |
| 类名不是名词 | Suggestion | PascalCase 名词 |
| 常量未大写 | Suggestion | UPPER_SNAKE_CASE |

### 错误处理

| 检查项 | 严重级别 | 说明 |
|--------|----------|------|
| 空 catch 块 | Critical | 至少记录日志 |
| 吞掉异常 | Critical | 正确传播或处理 |
| 过宽异常捕获 | Important | 捕获具体异常 |
| 缺少边界检查 | Important | 验证输入范围 |

## 反馈格式

### 严重级别定义

| 级别 | 含义 | 处理方式 |
|------|------|----------|
| **Critical** | 必须修复 | 阻止合并 |
| **Important** | 应该修复 | 建议修复后合并 |
| **Suggestion** | 建议改进 | 可选修复 |

### 反馈模板

```markdown
## 代码审查报告

**文件：** `path/to/file.py`
**审查人：** AI Assistant
**日期：** 2026-01-15

### Critical Issues (0)
无

### Important Issues (2)
1. **[行 45]** 函数过长，建议拆分
2. **[行 78]** 缺少空值检查

### Suggestions (1)
1. **[行 12]** 变量名 `x` 建议改为 `user_count`
```
