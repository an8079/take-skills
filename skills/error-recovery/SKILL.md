# 错误恢复 Skill

## When to Use

- 遇到编译错误
- 遇到运行时错误
- 遇到测试失败
- 用户说"修复错误"

## 错误分类

### 可自动修复的错误

| 错误类型 | 示例 | 修复策略 |
|----------|------|----------|
| 语法错误 | 缺少分号、括号不匹配 | 自动修复 |
| 导入错误 | 模块路径错误 | 自动修复 |
| 拼写错误 | 变量名拼写错误 | 自动修复 |
| 类型错误 | 类型不匹配 | 尝试修复 |
| 依赖缺失 | 缺少npm包 | 自动安装 |
| 配置错误 | 路径配置错误 | 自动修复 |

### 需要人工介入的错误

| 错误类型 | 示例 | 行动 |
|----------|------|------|
| 逻辑错误 | 业务逻辑不正确 | 暂停，询问用户 |
| 架构问题 | 设计不合理 | 暂停，询问用户 |
| 权限问题 | 无权限操作 | 暂停，提示用户 |
| 外部依赖 | 第三方API不可用 | 暂停，提示用户 |
| 安全问题 | 存在安全漏洞 | 暂停，提示用户 |

## 恢复流程

### 第一步：错误识别

**捕获错误信息：**
```
❌ 错误类型：[编译错误/运行时错误/测试失败]
❌ 错误位置：[文件:行号]
❌ 错误信息：[具体错误描述]
```

### 第二步：自动修复尝试

**修复策略（按优先级）：**

1. **语法错误** → 直接修复
2. **导入错误** → 修正路径
3. **类型错误** → 修正类型
4. **依赖缺失** → 安装依赖
5. **配置错误** → 修正配置

**修复尝试次数：最多3次**

```
🔧 尝试修复（第1次/共3次）

分析错误：缺少分号
修复方案：在第15行末尾添加分号
执行修复...
✅ 修复成功
```

### 第三步：验证修复

**修复后必须验证：**
1. 重新编译/运行
2. 运行相关测试
3. 检查是否引入新问题

```
✅ 验证通过
   - 编译成功
   - 测试通过
```

### 第四步：报告结果

**修复成功：**
```
✅ 错误已修复

错误：缺少分号
修复：在 src/utils/helper.js:15 添加分号
验证：编译通过
```

**修复失败（可自动修复）：**
```
⚠️ 自动修复失败（2次/共3次）

错误：类型不匹配
原因：user.name 可能为 undefined
尝试方案：
  - 方案1：添加类型检查
  - 方案2：使用可选链
  - 方案3：添加默认值

正在尝试下一个方案...
```

**修复失败（需要人工）：**
```
❌ 需要人工介入

错误类型：逻辑错误
错误描述：用户权限判断逻辑不正确

当前代码：
  if (user.role === 'admin') { ... }

问题：admin和super_admin权限处理相同

⚠️ 暂停自动执行，等待你确认修复方案
```

## 错误恢复策略

### 编译错误恢复

```
步骤：
1. 解析错误信息
2. 定位错误位置
3. 分析错误原因
4. 尝试自动修复
5. 重新编译验证
6. 报告结果
```

### 运行时错误恢复

```
步骤：
1. 捕获错误堆栈
2. 定位错误位置
3. 分析错误原因
4. 尝试自动修复
5. 重新运行验证
6. 报告结果
```

### 测试失败恢复

```
步骤：
1. 分析测试失败原因
2. 定位相关代码
3. 判断是代码问题还是测试问题
4. 修复代码或修正测试
5. 重新运行测试
6. 报告结果
```

## 自动恢复规则

### 最多尝试次数
- 同一错误：最多3次
- 不同错误：最多10次

### 暂停条件
- 错误类型为"需要人工介入"
- 修复次数超过上限
- 用户说"暂停"
- 修复后引入新错误

### 回滚策略
如果修复引入新错误：
1. 自动回滚到上一个 Checkpoint
2. 报告回滚结果
3. 提示需要人工处理

## 输出格式

### 错误发生

```
❌ 执行中断

错误类型：[类型]
错误位置：[位置]
错误信息：[信息]

🔧 正在尝试自动修复...
```

### 修复成功

```
✅ 修复成功

修复次数：1/3
修复方案：[方案描述]
验证结果：✅ 通过

继续执行任务...
```

### 需要人工

```
❌ 需要人工处理

错误：[描述]
尝试次数：3/3
原因：[原因]

【选项】
1. 回滚到上一个 Checkpoint
2. 跳过此任务
3. 由你手动处理
```

## 集成到半自动模式

在**编码实现阶段的自动执行中**：
- 遇到错误自动触发 error-recovery skill
- 最多尝试3次修复
- 修复成功继续执行
- 修复失败暂停并报告
