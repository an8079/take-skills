# cron-mastery — Cron调度大师

> 遇到cron表达式写不对？不知道哪个时间点触发？用这个skill直接给你生成+验证。

## 触发条件
- 用户提到cron、crontab、定时任务、调度
- 需要生成或解析cron表达式
- 不确定cron格式是否正确

## 工具列表
无特殊工具，主要靠分析 + 文档生成

## 技能参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| minutes | string | 否 | 分钟，默认* |
| hours | string | 否 | 小时，默认* |
| dayOfMonth | string | 否 | 日期，默认* |
| month | string | 否 | 月份，默认* |
| dayOfWeek | string | 否 | 星期，默认* |

## Cron格式说明

```
┌───────────── 分钟 (0-59)
│ ┌───────────── 小时 (0-23)
│ │ ┌───────────── 日期 (1-31)
│ │ │ ┌───────────── 月份 (1-12)
│ │ │ │ ┌───────────── 星期 (0-6，0=周日)
│ │ │ │ │
* * * * *
```

## 常用模式库

### 每分钟
```
* * * * *
```

### 每小时（第30分钟）
```
30 * * * *
```

### 每天凌晨3点
```
0 3 * * *
```

### 每4小时
```
0 */4 * * *
```

### 每周一凌晨6点
```
0 6 * * 1
```

### 每月1日凌晨0点
```
0 0 1 * *
```

### 工作日每天9点
```
0 9 * * 1-5
```

### 每30分钟（OpenClaw cron格式）
```
*/30 * * * *
```

## 验证规则
1. 数值范围必须在有效区间内
2. 星期用0-6，不要用7
3. 月份用1-12，不要混淆
4. */n 表示每n个单位触发一次

## 示例对话

**用户**: 帮我生成一个每周一三五下午3点执行的cron
**AI**: 
```
0 15 * * 1,3,5
```

**用户**: 这个cron是什么意思：0 */6 * * *
**AI**: 每6小时整点执行一次（0:00, 6:00, 12:00, 18:00）

## OpenClaw Cron特殊格式

OpenClaw支持自然语言调度，示例：
- `every 5 minutes` → 每5分钟
- `every 4h` → 每4小时
- `every day at 9am` → 每天9点
- `every Monday at 6` → 每周一6点

转换为标准cron：
```
*/5 * * * *
0 */4 * * *
0 9 * * *
0 6 * * 1
```
