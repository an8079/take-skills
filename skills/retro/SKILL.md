# Retro - 工程团队回顾与改进

## 概述

工程回顾是团队持续改进的关键。本技能提供系统化的回顾框架，从代码产出、团队协作、个人成长三个维度进行分析。

## 回顾类型

| 类型 | 频率 | 焦点 |
|-----|-----|-----|
| 周回顾 | 每周五 | 本周产出、下周计划 |
| Sprint 回顾 | 每个 Sprint 末 | 迭代复盘 |
| 项目回顾 | 项目结束时 | 整体经验教训 |
| 事故回顾 | 故障后 48h 内 | RCA + 改进 |

## 周回顾模板

### 1. 本周产出统计

```bash
# 获取本周提交统计
git log --since="monday" --oneline | wc -l
git log --since="monday" --format='%ae' | sort | uniq -c | sort -rn

# 获取 PR 统计
gh pr list --state=merged --since=$(date -d 'monday' +%Y-%m-%d) --json number,title,author,mergedAt

# 获取代码变更量
git diff --stat $(git log --oneline -1 --format=%H --since="monday")..HEAD
```

## Sprint 回顾模板

### 5. 速率分析

```bash
# Sprint 速率统计
echo "Sprint N 速率分析"
echo "计划: X 个故事点"
echo "完成: Y 个故事点"
echo "完成率: $(echo "scale=1; $Y * 100 / $X" | bc)%"

# 趋势图
echo "最近 6 个 Sprint 速率:"
for i in {1..6}; do
  sprint_points=$(git log --since="$((i*2))..$((i*2-2)) weeks" --oneline | wc -l)
  echo "Sprint $((7-i)): $sprint_points points"
done
```

## 团队贡献分析

```bash
# 个人贡献统计
git log --since="1 month" --format='%aN' | sort | \
  uniq -c | sort -rn | while read count name; do
  echo "$name: $count commits"
done

# 每人代码行数
for author in $(git log --format='%aN' | sort -u); do
  lines=$(git log --author="$author" --pretty=tformat: --numstat | \
    awk '{ add += $1; subs += $2 } END { printf "+%s, -%s", add, subs }')
  echo "$author: $lines"
done
```

## 回顾会议引导技巧

### 心理安全感
- 从"做得好的"开始，营造积极氛围
- 避免指责，聚焦系统性问题
- 鼓励所有人发言

### 聚焦行动
- 每个问题必须有对应的改进措施
- 改进措施必须有 Owner 和截止日期
- 定期跟踪改进项的落实情况

### 可视化
- 使用白板或工具可视化讨论结果
- 投票确定优先级
- 拍照保存讨论结果

## 总结

好的回顾会特点：
1. **坦诚** - 敢于说出真实想法
2. **具体** - 有具体的例子和数据
3. **建设性** - 聚焦于改进而非批评
4. **有行动** - 每个问题都有跟进措施
5. **被追踪** - 后续会回顾改进项进展
