# 反思 Skill

## When to Use

- 每个任务完成后
- 编码阶段自动执行
- 用户说"复盘"或"反思"

## 反思流程

### 第一步：结果检查

**检查任务是否达成目标：**
1. 对照验收标准检查完成情况
2. 检查代码是否能正常运行
3. 检查是否有遗漏的边界情况

### 第二步：质量自检

**必须检查的项目：**

| 检查项 | 检查内容 | 状态 |
|--------|----------|------|
| 功能完整性 | 所有功能点都已实现？ | ✅/❌ |
| 代码质量 | 无明显代码异味？ | ✅/❌ |
| 错误处理 | 异常情况有处理？ | ✅/❌ |
| 安全性 | 无安全漏洞？ | ✅/❌ |
| 性能 | 无明显性能问题？ | ✅/❌ |
| 可维护性 | 代码清晰易读？ | ✅/❌ |

### 第三步：问题识别

**发现的问题记录：**

```
⚠️ 发现问题：
1. [问题描述]
   - 影响：[影响范围]
   - 修复建议：[建议]

2. [问题描述]
   - 影响：[影响范围]
   - 修复建议：[建议]
```

### 第四步：自我修正

**对于可以立即修复的问题：**
1. 自动尝试修复
2. 验证修复结果
3. 更新 Checkpoint

### 第五步：总结输出

**反思总结格式：**

```
═══════════════════════════════════════
📝 任务反思总结：T-[N] [任务名]
═══════════════════════════════════════

✅ 完成情况：
- 验收标准达成：[X]/%

⚠️ 发现问题：[N]个
- 已修复：[N]个
- 待处理：[N]个

📊 改进建议：
- [建议1]
- [建议2]

═══════════════════════════════════════
```

## 自动触发时机

**任务完成后自动执行：**
1. 代码生成完成后
2. 测试执行完成后
3. 用户说"继续"之后

## 反思检查清单

```
□ 验收标准全部达成？
□ 代码能正常运行？
□ 有遗漏的边界情况？
□ 异常处理是否完善？
□ 是否有安全风险？
□ 性能是否可接受？
□ 代码是否清晰易读？
□ 有没有重复代码？
□ 命名是否规范？
□ 是否有调试代码未清理？
```

## 输出示例

**任务完成后自动显示：**

```
📝 任务反思：T-03 用户登录功能

✅ 达成：5/5 验收标准
⚠️ 问题：2个（已修复1个，待处理1个）

已修复：
- [修复] 登录失败后未返回具体错误信息

待处理：
- [建议] 可考虑添加登录失败次数限制

═══════════════════════════════════════
继续执行下一个任务...
═══════════════════════════════════════
```
