# 动态进度跟踪 Skill

## When to Use This Skill

- 每次会话开始时
- 每次任务完成时
- 每次阶段切换时
- 用户询问当前进度时
- 需要恢复中断的工作时

## 核心机制

### 1. 状态文件结构

```
memory-bank/
├── 项目进展.md           # 主状态文件（必须）
├── 当前任务.md           # 当前正在执行的任务
├── 任务历史.md           # 已完成任务记录
├── 技术决策.md           # 技术决策记录
└── 学习记录.md           # 用户纠正和学习内容
```

### 2. 必须记录的状态

| 字段 | 说明 | 示例 |
|------|------|------|
| 项目名称 | 当前项目名 | 我的待办应用 |
| 当前阶段 | 8 阶段中的当前阶段 | 编码实现 |
| 当前任务 | 正在执行的任务 ID | T-05 |
| 任务总数 | 总任务数 | 12 |
| 已完成数 | 已完成任务数 | 4 |
| 最后更新 | 最后更新时间 | 2026-02-22 10:30 |
| 阻塞问题 | 当前阻塞事项 | 无 |

### 3. 任务状态定义

| 状态 | 符号 | 说明 |
|------|------|------|
| 待开始 | ⬜ | 任务尚未开始 |
| 进行中 | 🔄 | 任务正在执行 |
| 已完成 | ✅ | 任务已完成 |
| 被阻塞 | 🔥 | 任务被阻塞 |
| 已跳过 | ⏭️ | 任务被跳过 |

## 工作流程

### 阶段一：会话开始时（必须执行）

**第一步：读取项目状态**

```
1. 读取 memory-bank/项目进展.md
2. 解析当前阶段、任务进度
3. 显示恢复信息给用户
```

**第二步：检查是否有未完成任务**

```
IF 有未完成的任务 THEN
    - 显示任务进度
    - 询问用户是否继续
ELSE
    - 显示阶段完成信息
    - 询问是否进入下一阶段
```

### 阶段二：任务执行中

**任务开始时**

```
1. 更新 memory-bank/当前任务.md
2. 设置任务状态为 "🔄 进行中"
3. 记录开始时间
```

**任务进行中（定期）**

```
1. 每完成一个子任务，更新进度百分比
2. 记录遇到的問題（如有）
```

**任务完成时（必须执行）**

```
1. 更新 memory-bank/项目进展.md
   - 将任务状态改为 "✅ 已完成"
   - 更新已完成数
   - 更新最后更新时间

2. 创建任务快照（可选）
   - 记录完成时间
   - 记录代码改动

3. 检查是否进入下一阶段
```

### 阶段三：阶段切换时

```
1. 确认当前阶段所有任务已完成
2. 更新 memory-bank/项目进展.md 中的阶段
3. 记录阶段切换时间
4. 询问用户是否进入下一阶段
```

## 强制规则

### 会话开始必须做的事情

```
1. 首先读取 memory-bank/项目进展.md
2. 显示当前状态：
   - 当前阶段
   - 当前任务
   - 进度百分比
   - 最后更新时间
3. 根据状态决定下一步操作
```

### 任务切换规则

```
1. 任务未完成不能切换到下一个任务
2. 阶段未完成不能切换到下一阶段
3. 除非用户明确说"跳过"或"暂停"
```

### 状态更新规则

```
1. 每次工具使用后，检查是否需要更新状态
2. 每次用户确认后，必须更新状态
3. 状态文件必须保持最新
```

## 检索机制

### 当需要检索历史时

```
1. 读取 memory-bank/项目进展.md 获取概览
2. 读取 memory-bank/任务历史.md 获取详细信息
3. 读取 memory-bank/技术决策.md 获取决策记录
4. 综合给出答案
```

### 当需要恢复工作时

```
1. 找到状态为 "🔄 进行中" 的任务
2. 读取 memory-bank/当前任务.md 获取详情
3. 从中断处继续
```

## 注意事项

1. **永远不要跳过状态读取** - 每次会话开始第一件事就是读 memory-bank
2. **永远不要跳过状态更新** - 每次任务/阶段完成必须更新
3. **状态优先于记忆** - 以 memory-bank 为准，不要依赖对话记忆
4. **及时处理阻塞** - 发现阻塞及时记录并询问用户

## 与其他 Skills 的关系

- **continuous-learning**: 学习记录单独管理
- **error-recovery**: 错误恢复时先读取状态
- **checkpoint**: 复杂任务使用 checkpoint 快照
