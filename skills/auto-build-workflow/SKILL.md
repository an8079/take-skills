# Auto-Build Workflow Skill - 全自动构建工作流

## 概述

这个 Skill 实现全自动项目构建，无需人工干预即可完成从需求到交付的完整流程。

## 触发条件

用户说出以下任一口令时触发：
- "自动开发"
- "全自动"
- "帮我做个[项目名]项目"
- "放开干"
- "auto-build"
- "start-auto"

## 前置要求

用户在 `docs/` 目录放入了参考资料：
- 项目需求文档
- API 定义文档
- 数据库结构文档
- 参考技术文档
- 业务流程文档

## 工作流程

### Step 1: 需求自动解析

```
┌─────────────────────────────────────────┐
│  扫描 docs/ 目录                         │
│  读取所有 .md 文件                       │
│  提取关键需求信息                        │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│  生成需求摘要                            │
│  - 项目名称                              │
│  - 核心功能                             │
│  - 技术栈                               │
│  - 业务流程                             │
└─────────────────────────────────────────┘
```

**执行脚本: `scripts/auto-analyze.js`**
```javascript
const fs = require('fs');
const path = require('path');

function analyzeDocs() {
  const docsDir = './docs';
  const files = fs.readdirSync(docsDir).filter(f => f.endsWith('.md'));

  let requirements = {
    name: '',
    description: '',
    features: [],
    techStack: [],
    apis: [],
    database: [],
    workflows: []
  };

  files.forEach(file => {
    const content = fs.readFileSync(path.join(docsDir, file), 'utf-8');
    // 智能解析内容
    parseContent(content, requirements);
  });

  return requirements;
}
```

### 并行执行策略

1. **分析阶段** - 串行（依赖文档）
2. **设计阶段** - 串行（依赖分析）
3. **实现阶段** - 可并行（多个 coder）
4. **审查阶段** - 串行（依赖实现）
5. **测试阶段** - 可并行（多个模块）
6. **交付阶段** - 串行

## 质量保证

### 自动化检查点

| 阶段 | 检查项 | 标准 |
|------|--------|------|
| 分析 | 文档完整性 | docs/ 有 >= 1 个 .md |
| 设计 | 规格文档 | spec.md 完整 |
| 实现 | 代码生成 | 所有任务完成 |
| 审查 | 问题数 | Critical=0, Important<=3 |
| 测试 | 覆盖率 | >= 80% |
| 交付 | 可运行 | docker-compose up 成功 |

### 错误恢复

1. **编译错误** - 自动修复或跳过
2. **测试失败** - 自动修复或调整
3. **审查失败** - 自动修复问题
4. **构建失败** - 尝试替代方案
