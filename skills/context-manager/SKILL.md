# 上下文管理 Skill

## When to Use

- 对话轮次超过20轮
- 上下文超过限制
- 用户说"总结"
- 任务切换时

## 上下文管理策略

### 1. 关键信息保留

**必须保留的信息：**

| 类型 | 内容 | 保留方式 |
|------|------|----------|
| 项目信息 | 项目名称、阶段、目标 | 永久 |
| 任务状态 | 当前任务、进度 | 永久 |
| 技术决策 | 架构选择、技术栈 | 永久 |
| 未完成任务 | 待完成任务列表 | 永久 |
| Checkpoint | 代码快照位置 | 永久 |

### 2. 可压缩的信息

**可以摘要的信息：**

| 类型 | 压缩方式 |
|------|----------|
| 对话历史 | 摘要关键点 |
| 代码改动 | 保留关键文件 |
| 错误尝试 | 只保留最终结论 |

### 3. 定期摘要

**触发摘要的时机：**

| 触发条件 | 动作 |
|----------|------|
| 20轮对话 | 执行摘要 |
| 超过限制 | 强制摘要 |
| 任务完成 | 可选摘要 |
| 用户要求 | 执行摘要 |

## 摘要流程

### 第一步：识别关键信息

**提取并保留：**

```
📋 项目信息
- 项目名：[项目名]
- 当前阶段：[阶段]
- 当前任务：[任务]

📋 任务进度
- 已完成：T-01, T-02, T-03
- 进行中：T-04
- 待完成：T-05, T-06

📋 技术决策
- 技术栈：React + Node.js + PostgreSQL
- 架构：前后端分离
- 认证：JWT

📋 Checkpoint
- 最新：Checkpoint #3 (T-03完成时)
```

### 第二步：压缩历史

**对话历史摘要：**

```
📜 对话历史摘要

【已完成】
- 访谈完成，确认了用户登录、注册、登出功能
- 规格文档已确认，包含5个API端点
- 计划生成，10个任务

【执行中】
- T-04：实现用户登录API
  - 已完成：路由定义、中间件
  - 待完成：业务逻辑、测试

【问题记录】
- 无阻塞问题
```

**输出摘要：**

```
═══════════════════════════════════════

- 技术栈：React + Node.js + PostgreSQL
- 认证方式：JWT

【已完成任务】
T-01: 项目初始化 ✅
T-02: 数据库设计 ✅
T-03: API设计 ✅

【进行中】
T-04: 用户登录API (60%)

【待执行】
T-05: 用户注册API
```

## 长对话处理

### 上下文接近限制

**警告阈值：80%**

```
⚠️ 上下文使用已达 80%

【选项】
1. 继续（可能会丢失早期信息）
2. 执行摘要（保留关键信息）
3. 暂停，等你确认
```

### 上下文超出限制

**自动触发摘要：**

```
📝 自动执行上下文摘要...

【保留的信息】
- 项目状态
- 任务进度
- 技术决策
- Checkpoint

【已摘要的内容】
- 早期对话历史
- 尝试过的错误方案

═══════════════════════════════════════
上下文已压缩，核心信息已保留
═══════════════════════════════════════
```

## 任务切换处理

### 任务间摘要

**每个任务完成后（可选）：**

```
📝 任务切换摘要：T-03 → T-04

【T-03 完成】
- 实现用户注册API
- 代码改动：3个文件
- 测试：3个用例通过

【T-04 准备】
- 用户登录API
- 预计改动：4个文件

═══════════════════════════════════════
```

### 阶段切换摘要

**每个阶段完成时（强制）：**

```
📝 阶段摘要：编码 → 测试

【编码阶段总结】
- 完成任务：7/10
- 代码改动：23个文件
- 创建Checkpoints：3个

【测试阶段准备】
- 待测试：23个文件
- 建议测试策略：先单元后集成

═══════════════════════════════════════
```

## 与 Memory Bank 集成

**摘要输出到：** `memory-bank/context-summary.md`

```markdown

## 项目信息
- 项目：[项目名]
- 阶段：[阶段]
- 任务：[任务]

## 关键信息

- 架构：[架构]
- 决策：[决策]

## 任务进度

- 进行中：[当前任务]
```
