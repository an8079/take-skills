# Deep Dive - 深度调查流水线

> 级别：L4（两阶段编排技能）
> 来源：oh-my-claude deep-dive | 适配：take-skills

---

## 角色定义

Deep Dive 执行**两阶段流水线**：先溯因调查（Trace），再精确定义需求（Deep Interview）。通过"三点注入"机制将Trace发现直接传入Interview阶段，避免上下文丢失。

## 触发词

- "deep dive"
- "deep-dive"
- "trace and interview"
- "深入调查"
- "查一下为什么"
- "根本原因是什么"

## 核心能力

- 并行三车道因果调查
- 三点注入机制（Findings × 3 → Interview初始化）
- 证据驱动的需求定义
- 执行桥接（不越权执行，只交接给后续技能）

## 两阶段架构

```
┌─────────────────────────────────────────────────┐
│               DEEP DIVE 流水线                   │
│                                                  │
│  ┌──────────────────┐     ┌──────────────────┐ │
│  │  阶段1: TRACE     │────→│  阶段2: INTERVIEW│ │
│  │  (自治)           │注入 │  (交互式)         │ │
│  │  根因调查 ×3车道  │     │  需求提炼         │ │
│  └──────────────────┘     └────────┬─────────┘ │
│                                      │           │
│                               ┌──────▼─────────┐ │
│                               │  阶段3: BRIDGE  │ │
│                               │  交接给执行技能 │ │
│                               └────────────────┘ │
└─────────────────────────────────────────────────┘
```

## 三点注入机制

Trace的发现通过以下3点注入到Interview：

| 注入点 | 内容 | 作用 |
|--------|------|------|
| **Enrichment** | Trace发现的根因和证据 | 丰富访谈起点 |
| **Context** | 系统相关区域和依赖 | 提供系统级背景 |
| **Seeds** | Trace无法自主回答的问题 | 播种初始问题 |

## 阶段1：Trace（并行3车道）

### 车道1：代码层

- 追踪问题涉及的文件和函数调用链
- 找错误源头（不是在症状处修）
- 标注关键代码位置

### 车道2：行为层

- 复现问题（运行测试、启动服务）
- 观察实际行为 vs 预期行为差异
- 记录可重现的步骤

### 车道3：知识层

- 查阅README、文档、代码注释
- 查git历史（问题从哪个commit引入）
- 查相关issue/PR背景

### 用户交互点

```
每车道先确认假设，再自治执行：
你: "我的假设是[X]，你同意吗？"
用户: [确认/否定/补充]
→ 基于确认的假设运行3车道调查
```

## 阶段2：Deep Interview

基于三点注入，交互式提炼需求。

### 规则

- **一次一问**：不批量提问
- **基于证据**：每个问题都有Trace发现支撑
- **探索未知**：问题聚焦Trace无法自主回答的部分

### 访谈结构

1. **问题是什么**（共识）
2. **为什么重要**（影响评估）
3. **成功标准是什么**（可测试）
4. **约束条件**（技术/时间/资源）
5. **优先级排序**（P0/P1/P2）

## 阶段3：执行桥接

不执行，只交接。输出：

```markdown
## 执行交接包

**问题定义**: ...
**根因摘要**: [来自Trace的3个关键发现]
**需求共识**: [来自Interview的5个要点]
**下一步**: 建议使用 [skill-name] 继续
**Artifact路径**: 
- trace-report.md
- interview-notes.md
```

## 禁止事项

- 不跳过Trace直接进入Interview（会丢失根因上下文）
- 不在Trace自治阶段中途打断用户
- 不在Bridge阶段自行执行（只交接）
- 不将Phase 1发现直接作为需求（需要Interview验证）
