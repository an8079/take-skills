## Quick Reference

### 规格编写原则

1. **面面俱到** — 编码时不应该有「这个没想到」
2. **可执行性** — 每个任务应该可以直接开始编码
3. **技术深度** — 不只说「做什么」，还要说「怎么做」
4. **风险前置** — 所有可能的坑都要提前识别

### 关键章节

- **功能清单** — 明确要做什么
- **验收标准** — 明确如何验证完成
- **数据模型** — 明确数据结构
- **接口定义** — 明确对外接口
- **非功能需求** — 明确性能、安全要求

```
# 电商订单系统规格文档

## 2. 功能规格

### 2.1 功能清单
| ID | 功能名称 | 优先级 | 复杂度 |
|----|----------|--------|--------|
| F-01 | 创建订单 | P0 | 中 |
| F-02 | 查询订单 | P0 | 低 |
| F-03 | 取消订单 | P1 | 中 |

### 2.2 用户故事

#### F-01: 创建订单

**用户故事：**
> 作为购物用户，我想要创建订单，以便购买商品

**验收标准：**
```gherkin
Scenario: 成功创建订单
  Given 用户已登录
  And 购物车中有商品
  When 用户点击"提交订单"按钮
  Then 订单创建成功
  And 订单状态为"待支付"
  And 购物车清空
```
```
