# 全自动构建功能使用指南

## 什么是全自动构建？

全自动构建是一种无需人工干预的开发模式。你只需要：
1. 把项目相关资料放到 `docs/` 目录
2. 说出触发口令
3. AI 会自动完成整个项目开发流程

## 触发口令

当你说以下任一语句时，将启动全自动构建：

| 口令 | 说明 |
|------|------|
| `自动开发` | 启动全自动项目构建 |
| `全自动` | 启动全自动项目构建 |
| `放开干` | 启动全自动项目构建 |
| `帮我做个XX项目` | 自动识别项目并构建 |
| `auto-build` | 启动全自动项目构建 |

## 准备工作

在启动全自动构建前，你需要把参考资料放到 `docs/` 目录。支持的文档类型：

### 1. 需求文档
文件名建议：`requirements.md` 或 `需求.md`

```markdown
# 项目名称：XXX系统

## 项目描述
[项目简介]

## 功能需求
1. 功能1
2. 功能2
3. 功能3
```

### 2. API定义
文件名建议：`api.md` 或 `接口.md`

```markdown
# API 接口定义

## 用户模块
- GET /api/users - 获取用户列表
- POST /api/users - 创建用户
- GET /api/users/{id} - 获取用户详情
```

### 3. 数据库结构
文件名建议：`database.md` 或 `数据库.md`

```markdown
# 数据库设计

## 用户表 users
- id: 主键
- username: 用户名
- email: 邮箱
- created_at: 创建时间
```

### 4. 技术栈偏好
文件名建议：`tech.md` 或 `技术栈.md`

```markdown
# 技术栈

后端：Python + FastAPI
数据库：PostgreSQL
前端：Vue 3
```

## 自动构建流程

当启动全自动构建后，AI 会自动执行以下步骤：

```
1. 📋 需求解析
   ↓ 扫描 docs/ 目录所有文档
2. 🏗️ 自动规格
   ↓ 生成 spec.md 架构设计
3. 💻 自动实现
   ↓ 调用 coder agent 编写代码
4. 🔍 自动审查
   ↓ 调用 reviewer + security-reviewer
5. 🧪 自动测试
   ↓ 调用 tester agent 生成测试
6. 🚀 自动交付
   ↓ 生成 Dockerfile + 测试页面
```

## 输出产物

全自动构建完成后，你会得到：

```
projects/[项目名]/
├── README.md           # 项目说明
├── docs/
│   └── spec.md        # 规格文档
├── src/               # 源代码
├── tests/             # 测试代码
├── deploy/            # 部署配置
│   ├── Dockerfile
│   └── docker-compose.yml
└── public/
    └── test.html      # API测试页面
```

## 常见问题

### Q: 需要准备多少文档？
A: 至少准备一个 `.md` 文件，描述你想要的项目即可。文档越多，理解越准确。

### Q: 可以指定技术栈吗？
A: 可以。在 docs/ 目录创建 `tech.md` 或在需求文档中说明技术偏好。

### Q: 构建时间多久？
A: 取决于项目复杂度，简单项目几分钟，复杂项目可能需要更长时间。

### Q: 如何中断全自动构建？
A: 直接说"暂停"即可停止。
