# 命令使用指南

## 概述

本文档详细说明 CLAUDE-STUDIO 中所有命令的用途、参数和使用方法。

---

## 核心命令（8 个）

### 1. /interview - 需求访谈

**用途：** 启动需求访谈流程，深入理解项目需求

**语法：**
```
/interview
```

**触发词：**
- `开始访谈`
- `需求访谈`

**工作流程：**
1. 调用 interviewer agent
2. 8 维度访谈
3. 生成需求摘要
4. 更新 Memory Bank

**输出示例：**
```
═══════════════════════════════════════
📋 需求访谈摘要
═══════════════════════════════════════

项目名称: [名称]
项目类型: Web 应用
核心功能: [功能列表]
目标用户: [用户群体]
技术约束: [约束条件]
...
```

---

### 2. /spec - 规格设计

**用途：** 设计系统架构，制定技术选型

**语法：**
```
/spec
```

**触发词：**
- `写规格`
- `生成规格`
- `规格设计`

**工作流程：**
1. 读取需求摘要
2. 设计系统架构
3. 选择技术栈
4. 定义接口契约
5. 生成 docs/spec.md

**输出文件：**
- `docs/spec.md`

---

### 3. /plan - 实现计划

**用途：** 将规格文档拆解为可执行的任务清单

**语法：**
```
/plan
```

**触发词：**
- `做计划`
- `创建计划`
- `拆解任务`

**工作流程：**
1. 读取 spec.md
2. 拆解任务
3. 分析依赖
4. 排序优先级
5. 输出任务清单

**输出内容：**
- 任务列表（ID、描述、依赖、优先级）
- 里程碑计划

---

### 4. /code - 编码实现

**用途：** 按计划实现功能代码

**语法：**
```
/code
```

**触发词：**
- `开始编码`
- `编码模式`

**工作流程：**
1. 读取任务清单
2. 按优先级实现
3. 编写测试（可选）
4. 更新 Memory Bank
5. 输出变更报告

**代码规范：**
- 遵循项目代码风格
- 添加必要的注释
- 处理错误边界

---

### 5. /tdd - TDD 模式

**用途：** 测试驱动开发模式

**语法：**
```
/tdd
```

**工作流程：**
1. 先写测试
2. 运行测试（失败）
3. 实现最小代码
4. 运行测试（通过）
5. 重构优化

**测试框架：**
- Jest (JavaScript/TypeScript)
- pytest (Python)
- JUnit (Java)

---

### 6. /test - 测试验证

**用途：** 生成并执行测试

**语法：**
```
/test
```

**触发词：**
- `做测试`
- `运行测试`

**工作流程：**
1. 分析代码覆盖率
2. 编写缺失测试
3. 执行测试套件
4. 生成测试报告

**测试类型：**
- 单元测试
- 集成测试
- E2E 测试

---

### 7. /review - 代码审查

**用途：** 审查代码质量和安全性

**语法：**
```
/review
```

**触发词：**
- `代码审查`
- `审查代码`

**审查维度：**
- 代码质量
- 最佳实践
- 安全漏洞
- 性能问题
- 命名规范

**输出：**
- 审查报告

---

### 8. /debug - 调试模式

**用途：** 分析错误并提供修复建议

**语法：**
```
/debug
```

**触发词：**
- `开始调试`
- `调试`

**工作流程：**
1. 收集错误信息
2. 分析堆栈跟踪
3. 定位问题代码
4. 提供修复方案

**使用示例：**
```

错误: TypeError: Cannot read property 'map' of undefined

```

## 增量开发命令（3 个）

### /import - 导入项目

**用途：** 导入现有项目并建立开发边界

**语法：**
```
/import [项目路径]
```

**选项：**
- 无参数：导入当前目录
- `--type=partial`：部分导入

**工作流程：**
1. 扫描项目结构
2. 检测技术栈
3. 生成架构报告
4. 创建边界配置

**输出：**
- `.claude/scope.yaml`

---

### /analyze - 分析项目

**用途：** 深入分析现有项目架构

**语法：**
```
/analyze [选项]
```

**选项：**
- `--quick`：快速扫描
- `--security`：安全分析
- `--full`：完整分析

**分析维度：**
- 项目结构
- 技术栈
- 入口点
- 模块关系
- 安全风险

---

### /scope - 开发边界

**用途：** 管理项目的开发边界

**语法：**
```
/scope <子命令>
```

**子命令：**
| 命令 | 说明 |
|------|------|
| `show` | 显示当前边界 |
| `add <path>` | 添加允许范围 |
| `remove <path>` | 移除允许范围 |
| `lock <path>` | 锁定区域 |
| `unlock <path>` | 解锁区域 |
| `reset` | 重置配置 |

**示例：**
```
/scope add src/features/user
/scope lock src/legacy
```

## 完整开发流程

### 新项目开发
```
开始访谈 → /interview
        → /spec
        → /plan
        → /code
        → /test
        → /review
```

### 增量开发
```
导入项目 → /import
分析架构 → /analyze
设置边界 → /scope
开始开发 → /code
```

## 阶段验证

每次阶段切换，系统会自动验证：

1. **前置条件**
   - 上一阶段在 phase-state.yaml 中标记为"已完成"
   - 必需的交付物存在

2. **状态检查**
   - Memory Bank 已更新
   - 关键字段已填写

3. **边界检查**
   - 开发的文件在允许范围内
   - 未修改锁定区域

## 常见问题

### Q: 如何跳过某个阶段？
A: 使用 `phase-cli.js jump <phase-id>` 跳过，但必须满足前置条件。

### Q: 阶段验证失败怎么办？
A: 检查：1) 上一阶段是否完成 2) 必需文件是否存在 3) Memory Bank 是否更新。

### Q: 如何查看当前阶段？
A: 运行 `node scripts/phase-manager.js status` 或查看 `memory-bank/项目进展.md`。
