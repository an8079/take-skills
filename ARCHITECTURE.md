# Claude Code 架构文档

> 版本：v1.0 | 更新日期：2026-01-24

---

## 概述

Claude Code 开发助手是一个基于 AI 的软件开发工作站，通过 Agent 架构和技能库系统，帮助开发者从需求分析到项目交付的全流程。

## Agent 架构

### Agent 职责矩阵

| Agent | 职责 | 输入 | 输出 | 依赖 |
|-------|------|------|------|------|
| interviewer | 需求访谈 | 用户需求 | 需求文档 | requirement-analysis |
| architect | 架构设计 | 需求文档 | 技术规格 | spec-writing |
| planner | 任务规划 | 技术规格 | 任务清单 | - |
| coder | 代码实现 | 任务清单 | 源代码 | coding-style |
| tester | 测试编写 | 源代码 | 测试用例 | testing, tdd-workflow |
| reviewer | 代码审查 | 源代码+测试 | 审查报告 | coding-style, security |
| security-reviewer | 安全审计 | 代码 | 安全报告 | security |
| devops | 构建部署 | 源代码 | 部署包 | deployment |
| optimizer | 优化分析 | 完整项目 | 优化建议 | performance |
| debug-helper | 问题调试 | 错误信息 | 解决方案 | - |

### Agent 交互模式

```
用户请求
    │
    ▼
┌──────────────────────────────────────┐
│         核心协调层                  │
│    (判断调用哪个 Agent/Skill)        │
└───────────────┬────────────────────┘
                │
     ┌──────────┼──────────┐
     ▼          ▼          ▼
   Agent1     Agent2     Skill1
     │          │          │
     └──────────┼──────────┘
                │
         Memory Bank
         (记录结果)
```

## Hooks 系统架构

### Hook 事件

| 事件 | 触发时机 | 用途 |
|------|----------|------|
| Start | 会话开始 | 初始化状态 |
| Stop | 会话结束 | 保存状态 |
| PreToolUse | 工具调用前 | 参数验证 |
| PostToolUse | 工具调用后 | 结果检查 |

### Hook 配置示例

```json
{
  "PreToolUse": [
    {
      "matcher": "tool == \"Edit\"",
      "hooks": [
        { "type": "validation", "check": "file_exists" }
      ]
    }
  ],
  "PostToolUse": [
    {
      "matcher": "tool == \"Edit\" && extension ~ /\\.(ts|js)$/",
      "hooks": [
        { "type": "quality", "check": "no_console_log" }
      ]
    }
  ]
}
```

## 扩展机制

### 添加新 Agent

1. 在 `agents/` 创建新文件
2. 定义 Agent 职责和工作流程
3. 更新 `CLAUDE.md` 的 Agent 表

### 添加新 Skill

1. 在 `skills/` 创建新目录
2. 创建 `SKILL.md` 文件
3. 定义技能元数据和内容
4. 更新 `CLAUDE.md` 的技能表

### 添加新 Command

1. 在 `commands/` 创建新文件
2. 定义命令的工作流程
3. 更新 `CLAUDE.md` 的命令表

## 质量保证

### 质量检查点

| 阶段 | 检查项 | 检查方式 |
|------|--------|----------|
| 访谈 | 8 维度完整度 | 完整度检查表 |
| 规格 | 技术可行性 | architect agent |
| 编码 | 代码风格 | coding-style + hooks |
| 测试 | 覆盖率 | tester agent |
| 审查 | 安全性 | security-reviewer |
| 交付 | 部署验证 | devops agent |

### 反馈循环

```
用户纠正 ───▶ 记录到学习记录.md
    │
    ▼ /reflect
    ├──▶ 全局规则 ───▶ 更新 CLAUDE.md
    ├──▶ 项目规则 ───▶ 更新项目 CLAUDE.md
    └──▶ 领域规则 ───▶ 更新对应 SKILL.md
```

## 部署架构

### Claude Code 插件模式

```
~/.claude-code/
├── plugins/
│   └── opencode-z/        # Claude Code 插件
│       ├── CLAUDE.md         # 主配置
│       ├── agents/           # Agents
│       ├── skills/           # Skills

│       └── ...
└── projects/
    └── project-a/            # 具体项目
        ├── CLAUDE.md         # 项目配置 (可选)
        └── ...
```

## 性能考虑

### 提示词优化

- 动态加载上下文，减少固定提示词长度
- 按需加载 Agent/Skill 内容
- 记忆系统仅加载相关部分

### 缓存策略

- Agent 响应缓存
- 技能内容缓存
- 规则匹配结果缓存

## 安全考虑

### 敏感信息保护

- `.env` 文件不纳入版本控制
- API 密钥使用环境变量
- Memory Bank 可配置加密

### 访问控制

- MCP 服务器访问控制
